# Requisitos
Requisitos do sistema da escola de idioma Blue Quill Language House

---

## Lista de Stakeholders (Personas)

- Admin

- Professores

- Responsáveis e Alunos

---

## Requisitos Funcionais(RF)

- O sistema deve permitir o cadastro e login de usuários em diferentes níveis (Administradores, professores, responsáveis e alunos).

- O sistema deve permitir que o usuário modifique os dados cadastrais.

- O sistema deve permitir que o administrador gerencie todos os usuários.

- O sistema deve permitir que o professor manipule a pauta de presença.

- O sistema deve permitir que o professor cadastre as notas.

- O sistema deve permitir que os alunos e responsáveis acessem as notas.

- O sistema deve permitir que os alunos e reponsáveis visualizem suas faltas.

- O sistema deve permitir que os alunos visualizem e façam as atividades.

- O sistema deve notificar os alunos sobre as atividades pendentes.

- O sistema deve salvar as atividades realizadas.

- O sistema deve permitir que o aluno visualize os horários disponíveis para as aulas de reposição e selecione um horário.

- O sistema deve enviar uma notificação sobre a data das reuniões bimestrais.

- O sistema deve permitir aos alunos e responsáveis acesso à informações financeiras e aos boletos para pagamento das mensalidades.

---
  
## Requisitos Não Funcionais(RNF)

- Compatibilidade: O sistema deve ser PWA (Rodar em diversas plataformas: computador e celulares). 

- Confiabilidade: O sistema deve estar disponível 99,5% do tempo.

- Segurança: O sistema deve criptografar todas as senhas.

- Performance: O sistema deve realizar a marcação de aulas de reposição em menos de 1 segundo.

- Acessibilidade: O sistema deve ser compativel com leitores de tela e possuir um modo de alto contraste.

---

## Requisitos Regras de Negócio(RN)

- Somente o admnistrador pode cadastrar usuários.

- O usuário deve estar logado para ter acesso as funcionalidades do sistema.

- Os alunos terão acesso somente as suas próprias informações.
